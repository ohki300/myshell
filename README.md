# ミニシェル

C言語で実装したUNIXライクなシェルです。パイプ、バックグラウンド実行、リダイレクション、ジョブ管理などの基本機能を備えています。

---

## 機能一覧

| 機能 | 説明 |
|------|------|
| コマンド実行 | `execvp` による外部コマンドの実行 |
| パイプ (`\|`) | 最大5段のパイプライン |
| バックグラウンド実行 (`&`) | コマンドをバックグラウンドで実行 |
| 出力リダイレクション (`>`) | 標準出力をファイルへ書き出し |
| ジョブ管理 | `jobs` / `fg` コマンドによる管理 |
| シグナル制御 | `SIGINT` / `SIGTSTP` 等の適切な処理 |

---

## ビルドと起動

```bash
gcc -o mysh main.c
./mysh
```

プロンプト `-> ` が表示されたらコマンドを入力できます。

---

## 使い方

### 通常のコマンド実行

```
-> ls -l
-> echo hello
```

### パイプ

コマンドを `|` で区切ります（最大5段）。トークンはスペース区切りで入力してください。

```
-> ls | grep .c
-> cat file.txt | sort | uniq
```

### バックグラウンド実行

コマンドの末尾に `&` を付けます。

```
-> sleep 10 &
[1] pid 1234
```

### 出力リダイレクション

`>` の後にファイル名を指定します。ファイルは新規作成または上書きされます。

```
-> ls > output.txt
-> echo hello > result.txt
```

### ジョブ一覧 (`jobs`)

バックグラウンドで動作中のプロセスを一覧表示します。

```
-> jobs
[1] pid 1234
[2] pid 5678
```

### フォアグラウンドに戻す (`fg`)

ジョブIDを指定してフォアグラウンドに移します。

```
-> fg 1
fg: pid 1234
```

### 終了

```
-> exit
```

または

```
-> quit
```

---

## 制限事項

- 入力コマンドの最大長は **100文字**
- パイプの最大段数は **5段**
- 入力リダイレクション (`<`) は未対応
- パイプ・リダイレクション記号は **スペースで区切る** 必要があります（例: `ls > out.txt`、`ls>out.txt` は不可）

## 内部構造

### データ構造

```
process 構造体（双方向リスト）
  ├── index   : ジョブID
  ├── pid     : プロセスID
  ├── pgid    : プロセスグループID
  └── state
        ├── pipe   : パイプ段数
        ├── bg     : バックグラウンドフラグ
        └── output : リダイレクト位置
```

バックグラウンドプロセスは双方向リスト（`head` ↔ `tail`）で管理されます。

### 処理フロー

```
入力受付 (input_command)
   ↓
スペースで分割・解析 (split_command_by_space)
   ↓
組み込みコマンド判定 (speecific_command_exec)
   ↓  該当しない場合
fork → execvp (create_process)
   ↓
フォアグラウンド: waitpid でブロック
バックグラウンド: リストに追加して即復帰
```

### シグナル処理

| シグナル | シェル側の動作 |
|----------|--------------|
| `SIGCHLD` | バックグラウンド終了検知 → リストから削除 |
| `SIGINT` / `SIGQUIT` / `SIGTSTP` | シェル自身は無視（子プロセスはデフォルトに戻す） |
| `SIGTTIN` / `SIGTTOU` | シェル自身は無視 |
